最简单的 molecule

点击
这个分子定义于页面内部，Molecule 会自动将定义提出，并删除定义元素。
  1. molecule 使用 molecule-def="包名.分子名" depends="依赖包,依赖包,..." 定义，任何 html 元素都可以变为 molecule包名中可以有 .，不写则默认为 noname
  2. 能动的 molecule 末尾调用 Molecule.create(Function) 函数，完成 js 和元素间的绑定，使其变为能响应的分子，称之为molecule构造函数
  3. molecule构造函数用 // MOLECULE_DEF, // MOLECULE_DEF_END 包围
  4. molecule 实例只要使用 molecule="包名.分子名" 即可创建
  5. 创建 molecule 实例不需要再执行任何脚本，像 $(createElement(tag)).attr('molecule', '分子名').appendTo(element) 或其它修改 DOM 的方式都可以生效
  6. 可以通过 Molecule.of(html element) 或 $(ele).molecule() 获取实例的 molecule 对象
  7. molecule 对象在实例化后，因为是调用 Molecule.create()创建，所以总是派生自 Molecule类，获得该类的方法
  8. 当 molecule 所附着的 html 元素删除时，molecule对象也伴随销毁
  9. 可以重载 dispose 函数 this.dispose= function(){} 进行收尾工作
  10. molecule 实例的 html 元素，可以有自己的 innerHTML，这段 html 会追加到 molecule实例化后的 html 末尾
  11. 也可在 molecule 定义中插入 <!-- {INNER_HTML} --> ，作为实例的插入点
  12. 实例中的script脚本，会在置入 html 后再执行，所以实例可以放心的使用 molecule 对象，但是应该检查 Molecule.processing == true，避免在 molecule 未启动时执行
  13. molecule 实例自定义的 html attribute，在实例化后仍然保留, 通过以上两项可以进行html层面的重载
  14. 在 molecule 定义中夹杂 css 脚本，可以使css样式、html元素、能动函数混为一体
  15. molecule 构造函数可以通过 this.$el 以 jQuery 方式访问html元素，也可以使用 this.el 访问原 html 元素
  16. 实例的 html 元素可以通过设置 param1="" param2="" data-option="json" 提供初始化参数，以实现参数化
  17. 实例显然也可以重载 css 样式表，因为实例自身的 css 样式表总是能覆盖模板的样式表，但是这种覆盖也会影响其它实例，自己想想办法(见下面的例子)
  18. 实例的 class 和 style 属性，可以使用头字符 +，表示延续 molecule 模板的 css 或 style
  19. 实例可以嵌套入其它 molecule，形成更大元素，这种元素可以加上 molecule-def 导出为新的大分子，也就是说分子可以聚合为更大的分子
  20. 不要在 MOLECULE_DEF 中放置公用函数，这些函数不实例化是不会调用的，应将它们定义于额外的文件中
